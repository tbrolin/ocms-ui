# Web components - short primer/recap 1. `<link rel="import" href="imported.html">` New link relationship - import. Imports the specified html document into current document. http://w3c.github.io/webcomponents/spec/imports/ 2. `document.registerElement(name, options)` example: ```javascript // default: var MyElement = document.registerElement('my-element'); // equivalent to: // document.registerElement('my-element', { // 'prototype': HTMLElement.prototype // }); var myElement = new MyElement(); document.appendChild(myElement); // but it doesn't stop there: var MyCoolElement = document.registerElement('my-cool-element', { 'prototype': Object.create(HTMLElement.prototype, { 'createdCallback': function () { console.log('Created element!')}, 'attachedCallback': function () { console.log('Element attached to DOM!')}, 'detachedCallback': function () { console.log('Noo, why you not like me?')}, 'attributeChangedCallback': function () { console.log('One of my attributes was modified!')} }); }); // You can even extend existing elements var SpecialButton = document.registerElement('special-button', { 'prototype': Object.create(HTMLButtonElement.prototype), 'extends': 'button' }); // used in html like: // <button is="special-button"> ``` https://developer.mozilla.org/en-US/docs/Web/API/Document/registerElement 3. `<template>` element: Holds html not rendered. https://developer.mozilla.org/en/docs/Web/HTML/Element/template 4. Shadow DOM purpose: Isolate a DOM fragment from a 'hosting' DOM. https://developer.mozilla.org/en-US/docs/Web/Web_Components/Shadow_DOM # My little framework I wanted a way to ajax load components rather than relying on the link element because the imported resource must be .html and hence served by some web server somewhere (also it suffers from CORS restrictions which may or may not be a problem).
